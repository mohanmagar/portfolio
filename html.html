HTML (HyperText Markup Language)

Definition

HTML is the foundational language used to create and structure the content on a webpage. I understand it as the skeleton of a website. It uses a system of "tags" to define different types of content, such as headings, paragraphs, images, and links, which a web browser then interprets and displays to the user.

Types of HTML Tags

During the module, I learned that not all tags are the same. They can be grouped by their purpose, which helps in writing better, more meaningful code:

  • Semantic Tags: These tags provide context and meaning to the content they contain. Examples include <header>, <footer>, <nav>, <main>, <article>, and <section>. Using them is important for accessibility (helping screen readers understand the page layout) and for SEO (helping search engines index the site correctly).
  • Structural Tags: These are the fundamental tags that form the basic structure of any HTML document, such as <html>, <head>, and <body>.
  • Formatting Tags: These are used to format text, like <strong> for making text bold and important, <em> for italicizing text, and <blockquote> for defining a section that is quoted from another source.
  • Media and Link Tags: These are used to embed content or link to other resources. The most common are <img> for images, <a> for hyperlinks, and <video> for embedding videos.
  • Form Tags: These are essential for user interaction. Tags like <form>, <input>, <label>, and <button> allow us to build forms to collect user data.
  • Table Tags: For displaying data in a structured, tabular format, we use <table> along with <thead>, <tbody>, <tr>, <th>, and <td>.

Working Example Explanation

My live HTML example is a single webpage that demonstrates my understanding of how to use these tags to build a well-structured and semantic document.

  • Semantic Layout: The page is structured using <header>, <nav>, <main>, and <footer> to define the main sections of the page. This is a better practice than using generic <div> tags for everything, as it creates a more meaningful document structure.
  • Accessible Form: The example includes a contact form that uses <label> tags correctly associated with their <input> fields. This is crucial for accessibility. I have also used specific input types like type="email" for better user experience on mobile devices and the required attribute for basic client-side validation.
  • Structured Data: I've included a <table> with a <thead> and <tbody> to show how to correctly structure tabular data, which is important for both readability and accessibility.

This example shows that I can create a solid, accessible, and well-structured foundation for any webpage.
